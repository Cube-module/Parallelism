## Описание

Генерируется массив из 10⁷ элементов со значениями синуса от 0 до 2π. 
Вычисляется сумма всех элементов и выводится в терминал.

## Точность Float

-0.0277862

## Точность Double

6.27585e-10

## Причины 
При использовании типа float вычислительная ошибка оказывается значительно больше, чем при использовании double. Это связано с тем, что float хранит примерно 7 значащих цифр, тогда как double — около 15–16.

Из-за меньшей точности представления чисел каждое арифметическое действие в float сопровождается более грубым округлением. При большом количестве операций (например, при суммировании миллионов значений) эти погрешности накапливаются, что приводит к заметному отклонению результата от теоретического значения.

Особенно сильно ошибка проявляется при сложении большого количества чисел, когда часть значений может теряться из-за ограниченной точности (например, при добавлении малых чисел к большим).

В задачах машинного обучения, несмотря на меньшую точность, тип float используется широко, так как нейронные сети решают приближённые задачи, устойчивы к небольшим численным погрешностям, а использование float позволяет ускорить вычисления и снизить потребление памяти.

## Сборка и запуск

### Требования
- CMake 3.14+
- Компилятор C++17 (MinGW или MSVC)

### Сборка

```bash
# Клонирование репозитория
git clone <url-репозитория>
cd sinus_hw_1

# Конфигурация с double (по умолчанию)
cmake -B build

# Или конфигурация с float
cmake -B build -DUSE_FLOAT=ON

# Сборка
cmake --build build
